iOS-rörligt är ett grindbeslut: duger klippet? Godkänn så det finns hemma. Ingen trim, ingen grafik — det är desktopens och Resolves jobb.
Proxy från telefonen

Pass om det finns
"Del av 100 m dam · Final". Utan pass kopplas klippet till jobbet/matchen — samma regel som stillbild.
Duger-för i klartext
Längd vägs mot kanalgränsen direkt ("Reel ✓"), så Stig ser på plats om klippet håller.
Förkasta = göm, inte radera
Ett förkastat klipp göms ur flödet men mastern behålls på Macen — icke-destruktivt, som NEF. Går att ångra.

08 SVAR PÅ F1–F7
F1Namn & hem
Rörligt / klipp. Egen panel i railen + ett litet spår där bild redan bor (Leverera/Publicera) — inte en till flik i den redan långa Publicera-panelen.
F2Granskning
1b som lista, 1a som detalj. Snabbplock-rutnät med hover-preview för urval; granskningsbord med scrub + enkel in/ut bakom dubbelklick. Djuptrim = Resolve.
F3Blandade paket
Fotografen väljer per paket. Blandat tillåts (plan-listan sorterar karusell/reel/story); vill man hålla listan enkel läggs rörligt som eget paket. Valet ligger på paketet.
F4Async reel
D9-språket, i publiceringskön. 🔵 inskickat → 🟡 bearbetas → 🟢 publicerat, 🔴 fel. Pågår-tillståndet överlever panelbyte; fel efter stängd panel visas i kön med "gör om".
F5Grafik
Frusen som standard, kan spela. Samma chips + samma Pillow-överlägg som stillbild. Placering väljbar per klipp (hela / första 3 s / endcard).
F6Leverans
R2 + signerad länk. Utgångsdatum, status i dpt.db ("levererat 14:20, öppnat 2 ggr"). Kundsida bär DPT-signaturen; fotografen kan kopiera/förlänga/återkalla.
F7Ingest-tillstånd
En prick, inte en statuslista. Liten färgprick på klippet (oproxat/proxas/redo/master saknas/Resolve). Detaljen syns på hover/i granskningsbordet — aldrig teknisk text i ansiktet på Stig.
